# Maze

## Краткое описание управления

Игрок перемещается клавишами `WASD`. Направление движения считается относительно камеры. Ускорение включается при удержании `Left Shift`.

## Краткое описание архитектуры

`Main` - точка входа gameplay-логики. Создаёт `GameStateMachine`, регистрирует состояния, подписывается на события победы, поражения и рестарта.

`GameStateMachine` и `IState` - простая машина состояний игры. Переключает состояния через `Enter()` / `Exit()`.

`GameContext` - контейнер ссылок на основные сервисы и UI: `RoundService`, счётчик бриллиантов, конфиг спавна, стену выхода, `WinTrigger`, `WinView`, `LoseView`.

`RoundService` - управляет подготовкой раунда: очищает старого игрока, спавнит нового, сбрасывает счётчик бриллиантов, запускает спавн бриллиантов, инициализирует врагов, включает/выключает акторов раунда.

`LoadingState` - прячет экраны результата, подготавливает раунд и сразу переводит игру в `GameplayState`.

`GameplayState` - основной игровой режим. Включает игрока и врагов, закрывает выход, подписывается на сбор бриллиантов и открывает выход, когда собрано нужное количество.

`WinState` - состояние победы: выключает движение/врагов и показывает `WinView`.

`LoseState` - состояние поражения: выключает движение/врагов и показывает `LoseView`.

## Где логика победы/поражения

Победа: `GameplayState` открывает выход, когда `DiamondPickedCounter.DiamondPickedCount >= DiamondSpawnConfig.Count`. После этого `WinTrigger` ловит вход игрока в триггер выхода и вызывает событие `PlayerReachedExit`. `Main.OnPlayerReachedExit()` переводит игру в `WinState`.

Поражение: `EnemyPatrolChase` вызывает `PlayerCaught`, когда активный враг касается игрока. `EnemiesService` пробрасывает это событие выше. `RoundService` пробрасывает событие `PlayerCaught`. `Main.OnPlayerCaught()` переводит игру в `LoseState`.

## Где логика сбора бриллиантов

`Diamond` - реагирует на `OnTriggerEnter` игрока, увеличивает счётчик и уничтожает объект бриллианта.

`DiamondPickedCounter` - хранит количество собранных бриллиантов и отправляет событие `OnDiamondPicked`.

`DiamondsSpawner` - спавнит бриллианты на NavMesh, учитывая количество из `DiamondSpawnConfig`, дистанцию между бриллиантами и отступ от края NavMesh.

`DiamondFactory` - создаёт бриллиант и передаёт ему ссылку на `DiamondPickedCounter`.

`DiamondView` - показывает прогресс сбора в UI.

## Где логика врагов

`EnemiesService` - управляет массивом врагов: инициализирует их целью-игроком, включает/выключает, подписывается на событие поимки игрока.

`EnemyPatrolChase` - основная логика врага. Враг патрулирует случайные точки в своей зоне через `NavMeshAgent`, замечает игрока в радиусе detection, переходит в chase-состояние, преследует игрока, теряет цель за пределами lose-radius и возвращается к патрулю. При столкновении с игроком вызывает событие поражения.
